# Google BigQuery 最新機能 Deep Research レポート (2025〜2026)

> 調査期間: 2025年1月1日〜2026年3月現在
> 調査実施日: 2026年3月9日

---

## 1. 主要な新機能リスト

### AI・機械学習機能

| 機能名 | 概要 | リリース時期・ステータス |
|--------|------|--------------------------|
| **BigQuery AI 関数 (AI.GENERATE / AI.GENERATE_TABLE)** | テキスト・画像・動画・音声・ドキュメントを入力として受け取り、抽出・翻訳・要約・感情分析などのAI/MLタスクを SQL から直接実行可能 | 2025年4月 (Google Cloud Next '25) にGA昇格 |
| **AI.EMBED / AI.SIMILARITY** | Gemini埋め込みモデルおよび13,000以上のOSSモデルをBigQuery内で直接使用可能。AI.SIMILARITYは埋め込み生成と類似度計算を1ステップで実行 | 2025年 (Preview → GA) |
| **AI.IF / AI.CLASSIFY / AI.SCORE** | 自然言語で記述した条件でデータフィルタリング・分類・スコアリングを行うマネージドAI関数。プロンプトチューニング不要 | 2025年 (Preview) |
| **BigQuery AI クエリエンジン** | 従来のSQL処理にGeminiによるリアルタイム推論・外部知識注入を組み合わせ、構造化データと非構造化データを同時処理 | 2025年 (Preview) |
| **TimesFM フォーキャスティングモデル (AI.FORECAST)** | Google Researchの1,000億以上の実世界タイムポイントで学習済みの事前訓練済み時系列予測モデル。AI.FORECASTで呼び出し可能 | 2025年4月 (Google Cloud Next '25) |
| **BigQuery ナレッジエンジン** | スキーマ関係・テーブル説明・クエリ履歴をGeminiが分析し、メタデータ自動生成・データ関係モデリング・ビジネス用語集の推薦を実現 | 2025年 (Preview) |
| **マルチモーダルテーブル** | 構造化データと非構造化データ（画像・動画など）を1つのテーブルで格納・分析可能 | 2025年4月 (Google Cloud Next '25 / Preview) |

### データエンジニアリング・エージェント機能

| 機能名 | 概要 | リリース時期・ステータス |
|--------|------|--------------------------|
| **データエンジニアリングエージェント** | データパイプライン構築、データ変換、異常検知による品質管理、メタデータ自動生成を支援するAIエージェント | 2025年4月 (GA) |
| **データサイエンスエージェント** | Colabノートブック内に組み込まれ、特徴量エンジニアリング・モデル選択・学習を自動化。Apache Spark / PySpark キーワードのプロンプト対応 | 2025年 (GA) |
| **BigQuery リモート MCP サーバー** | Model Context Protocol (MCP) を通じてLLMエージェントがBigQueryのデータ関連タスクを直接実行できるマネージドサーバー。LangGraph・Claude Code・Cursor IDEなどと連携可能。OAuth 2.0 + IAM + Model Armorで保護 | 2025年12月10日 (Preview)、2026年1月以降 (拡充) |
| **会話型アナリティクス (Conversational Analytics)** | 自然言語でデータを分析し、SQL生成・実行・可視化まで一貫して行うインテリジェントエージェント。BigQuery StudioとLookerのセマンティックモデルに対応 | 2025年 (Preview / Public Preview) |

### BigQuery Studio 強化

| 機能名 | 概要 | リリース時期・ステータス |
|--------|------|--------------------------|
| **データキャンバス (Data Canvas)** | グラフィカルインターフェースでデータの探索・変換・クエリ・可視化を実現するノーコード型データ分析ツール | 2025年 (Preview) |
| **データ準備 (Data Preparation)** | スキーマ管理・データ品質自動化・合成データ生成をGeminiが支援するAIファーストのデータ準備ソリューション | 2025年 (Preview) |
| **データセットインサイト** | テーブル間の関係グラフを生成し、クロステーブルクエリや自動データセット要約、分析クエリ候補を提案 | 2025年 (GA) |
| **Gemini コードアシスタント (SQL/Python)** | BigQuery Studioのクエリエディタ・ノートブックエディタでSQL/Pythonコードを自動生成・補完 | 2025年 (GA) |
| **リファレンスパネル** | クエリ・ノートブックエディタからテーブル・スナップショット・ビュー・マテリアライズドビューのスキーマを即時プレビュー | 2025年 (GA) |

### パフォーマンス・ランタイム改善

| 機能名 | 概要 | リリース時期・ステータス |
|--------|------|--------------------------|
| **BigQuery 高度なランタイム (Advanced Runtime)** | クエリ実行時間とスロット使用量を大幅改善する新ランタイム。2025年9月15日〜2026年初頭にかけて全プロジェクトのデフォルトに移行 | 2025年 (GA) |
| **短いクエリ最適化 (Short Query Optimization)** | 高度なランタイムの拡張機能。一部のクエリをシングルステージで実行しレイテンシとスロット使用量を削減 | 2025年 (Preview) |
| **ワークロード管理オートスケーラー強化** | 2024年7月発表の機能が全ユーザーへ展開完了 | 2025年2月25日 (GA) |
| **ベクター検索パーティションインデックス** | ベクター検索クエリのコストを大幅削減するパーティションインデックス機能 | 2025年 (Preview) |

### データ統合・クロスクラウド機能

| 機能名 | 概要 | リリース時期・ステータス |
|--------|------|--------------------------|
| **クロスクラウドジョイン** | Google Cloud と BigQuery Omni リージョン間でのクロスクラウドクエリ結合が可能に | 2025年 (GA) |
| **Delta Lake サポート (S3/Azure)** | Amazon S3 および Azure 上の Delta Lake フォーマットテーブルをネイティブクエリ可能。データ移動不要 | 2025年 (GA) |
| **BigLake Iceberg テーブル** | 高スループットストリーミング・自動再クラスタリング・Vertex AI ネイティブ統合を備えた Apache Iceberg テーブル | 2025年 (GA) |
| **Continuous Query (連続クエリ)** | リアルタイムストリーミングデータに対して継続的にSQL処理・ML推論を実行。Spanner へのリアルタイムストリーミングも対応 | 2025年 (GA) |

### SQL 言語・開発機能

| 機能名 | 概要 | リリース時期・ステータス |
|--------|------|--------------------------|
| **パイプ構文 (Pipe Syntax) GA** | `\|>` 演算子でSQLを順次記述できる拡張SQL構文。SELECT・WHERE・JOIN等を逐次チェーン可能 | 2025年2月4日 (GA) |
| **BigQuery Governance Center** | データリネージ・ポリシー適用・データおよびモデルの異常検知を統合したガバナンス基盤 | 2025年4月 (GA) |
| **データセット削除取り消し (Undelete)** | タイムトラベルウィンドウ内であれば削除されたデータセットを削除前の状態に復元可能 | 2025年 (GA) |

| イベント | 内容 |
|----------|------|

## 2. 重要なポイントの要約

1. **BigQuery は「AI データプラットフォーム」へ進化**: Google Cloud Next '25（2025年4月）を最大の転換点として、AI.GENERATE・AI.EMBED・AI.SIMILARITY・AI.FORECASTなどのSQL関数で、コードを書かずにGeminiの能力をデータ処理パイプラインに組み込めるようになった。

2. **MCP サーバーによるエージェント連携が最重要トレンド**: 2025年12月10日にリリースされた BigQuery リモート MCP サーバーは、Claude Code・LangGraph・Cursor IDE などのLLMエージェントが BigQuery に直接接続できる標準インターフェースを提供。

3. **オープンフォーマット対応の大幅強化**: Apache Iceberg・Delta Lake のネイティブサポートが相次いでGAとなり、クロスクラウドジョイン（AWS・Azure・GCP間）も実現。

4. **Legacy SQL の終焉が正式決定**: 2026年6月1日には事実上の廃止へ。既存のLegacy SQLワークロードを持つ企業は今すぐGoogleSQLへの移行計画が必要。

5. **ガバナンスとリアルタイム処理の成熟**: BigQuery Governance Center（GA）・Continuous Query（GA）により、分析・運用・ガバナンスを単一プラットフォームで完結させる機能が揃った。
